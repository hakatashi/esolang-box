# AGENTS.md

This file provides guidance to coding agents when working with code in this repository.

## What This Project Is

esolang-box is a collection of Docker images for 200+ esoteric (and non-esoteric) programming languages, published to Docker Hub under the `esolang/` namespace (e.g., `esolang/brainfuck`, `esolang/ruby`). Each image provides a standardized interface: run `script /code/filename` with stdin for input and get stdout for output.

## Setup

```bash
bundle install   # Install Ruby dependencies (requires Ruby 3.0+)
```

Building requires Docker with BuildKit support (`docker buildx bake`).

## Key Commands

```bash
# Regenerate docker-bake.hcl from boxes/*/box.yaml (do this after editing box.yaml files)
bundle exec ruby scripts/build.rb

# Build all Docker images
docker buildx bake

# Build a single language image
docker buildx bake <language>   # e.g., docker buildx bake brainfuck-bfi

# Run the full test suite
bundle exec rspec

# Run tests for a specific language
bundle exec rspec -t box_id:brainfuck-bfi

# Run single test case
bundle exec rspec -t case_id:python3-hello

# Run tests in parallel
bundle exec parallel_rspec spec/
```

## Architecture

### Configuration-Driven Design

The project is driven by per-language `boxes/<language>/box.yaml` files. **Never edit [docker-bake.hcl](docker-bake.hcl) directly** — it is auto-generated by `ruby scripts/build.rb` from the individual `box.yaml` files. The schema for `box.yaml` is defined in [schemas/box.yaml.schema.json](schemas/box.yaml.schema.json).

### Image Hierarchy

Images inherit from one another using Docker's FROM mechanism. The hierarchy is defined through the `parent` field in each `box.yaml`:

```
esolang/base (Alpine Linux 3.19)
└── esolang/build-base (adds build tools: gcc, make, etc.)
    ├── esolang/ruby → esolang/golfscript, esolang/brainfuck, ...
    ├── esolang/python → esolang/evil, esolang/starry, ...
    ├── esolang/java → esolang/lolcode, ...
    └── ...
esolang/ubuntu-base (Ubuntu-based)
└── esolang/crystal, esolang/ada, ...
```

### Adding a New Language

1. Create a directory at `boxes/<language>/` containing:
   - `Dockerfile` — FROM the parent image, installs the language
   - `script` — shell wrapper that invokes the interpreter
   - `box.yaml` — metadata and test definitions (see schema below)
   - `assets/` — test program files referenced by `box.yaml`
2. Run `ruby scripts/build.rb` to regenerate docker-bake.hcl

### box.yaml Format

Each language directory contains a `box.yaml` file with the following fields:

```yaml
name: Language Display Name          # Display name (optional)
link: "https://example.com"          # Official language URL (optional)
parent: ruby                         # Parent image name (optional, for hierarchy)
disabled: false                      # Exclude from builds (default: false)
disabled_after: "2.3.0"             # Version after which the language was disabled (optional)
skip: "reason"                       # Reason for skipping tests (used when disabled: true)
disasm: true                         # Enable disassembly support via /root/disasm (default: false)
container_timeout: 180               # Container wait timeout in seconds (default: 180)
tests:                               # Test cases keyed by name
  hello:
    file: hello.bf                   # Asset filename relative to boxes/<lang>/assets/
    expected: "Hello, World!\n"      # Expected stdout output
  cat:
    file: cat.bf
    stdin: "test input"              # Input to feed via stdin (optional)
    expected: "test input"
    skip: "reason"                   # Skip this individual test (optional)
credits:                             # Credits keyed by asset filename (optional)
  hello.bf: "[Author](https://example.com)"
```

### Standard Script Interface

Every language image must have `/bin/script` (provided by the base image) and `/root/script`. The `/root/script` receives the code file path as `$1` and stdin from the user. Environment variables:
- `STRACE_OUTPUT_PATH` — if set, enables strace tracing (requires `--cap-add=SYS_PTRACE`)
- `ENABLE_STRACE` — enables strace in tests

### Disassembly Support

Languages can support a disassembly mode by setting `disasm: true` in `box.yaml` and providing `/root/disasm`. The test suite will automatically test this if a disasm test is configured in `box.yaml`.

### Testing

Tests in [spec/boxes_spec.rb](spec/boxes_spec.rb) read `box.yaml` from each language directory and run Docker containers. Each test case in `box.yaml` defines:
1. The source file (from `boxes/<lang>/assets/`)
2. Optional stdin input
3. Expected stdout output

### Obsolete Languages

Languages that are no longer maintained can be marked with `disabled: true` and `disabled_after: "<version>"` in their `box.yaml`. These appear in the generated README under "Obsolete languages".

### Publishing

After building, Docker Hub READMEs are updated via `ruby push.rb` (requires a `TOKEN` env var with a Docker Hub JWT). The README content is generated from [box_readme.md.erb](box_readme.md.erb) and pulls Hello World examples from the test assets.
