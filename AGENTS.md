# AGENTS.md

This file provides guidance to coding agents when working with code in this repository.

## What This Project Is

esolang-box is a collection of Docker images for 200+ esoteric (and non-esoteric) programming languages, published to Docker Hub under the `esolang/` namespace (e.g., `esolang/brainfuck`, `esolang/ruby`). Each image provides a standardized interface: run `script /code/filename` with stdin for input and get stdout for output.

## Setup

```bash
bundle install   # Install Ruby dependencies (requires Ruby 3.0+)
```

Building requires [dobi](https://github.com/dnephin/dobi) v0.10+ and Docker.

## Key Commands

```bash
# Regenerate dobi.yaml from boxes.yml (do this after editing boxes.yml)
ruby build.rb

# Build all Docker images
dobi

# Build a single language image
dobi image-<language>   # e.g., dobi image-brainfuck

# Run the full test suite
bundle exec rspec

# Run tests for a specific language
bundle exec rspec -e "brainfuck"

# Run tests in parallel
bundle exec parallel_rspec spec/
```

## Architecture

### Configuration-Driven Design

The project is driven by [boxes.yml](boxes.yml), the master configuration file that defines all languages and their hierarchy. **Never edit [dobi.yaml](dobi.yaml) directly** — it is auto-generated by `ruby build.rb` from boxes.yml.

### Image Hierarchy

Images inherit from one another using Docker's FROM mechanism. The hierarchy is defined in boxes.yml through nesting:

```
esolang/base (Alpine Linux 3.19)
└── esolang/build-base (adds build tools: gcc, make, etc.)
    ├── esolang/ruby → esolang/golfscript, esolang/brainfuck, ...
    ├── esolang/python → esolang/evil, esolang/starry, ...
    ├── esolang/java → esolang/lolcode, ...
    └── ...
esolang/ubuntu-base (Ubuntu-based)
└── esolang/crystal, esolang/ada, ...
```

### Adding a New Language

1. Add an entry to [boxes.yml](boxes.yml) nested under the appropriate parent image
2. Create a directory at `boxes/<language>/` containing:
   - `Dockerfile` — FROM the parent image, installs the language
   - `script` — shell wrapper that invokes the interpreter
3. Add test assets in `spec/assets/`:
   - `hello.<ext>` — prints "Hello, World!" (required)
   - `cat.<ext>` — reads stdin and echoes it back (optional)
4. Run `ruby build.rb` to regenerate dobi.yaml

### Standard Script Interface

Every language image must have `/bin/script` (provided by the base image) and `/root/script`. The `/root/script` receives the code file path as `$1` and stdin from the user. Environment variables:
- `STRACE_OUTPUT_PATH` — if set, enables strace tracing (requires `--cap-add=SYS_PTRACE`)
- `ENABLE_STRACE` — enables strace in tests

### Disassembly Support

Languages can support a disassembly mode by setting `_disasm: true` in boxes.yml and providing `/root/disasm`. The test suite will automatically test this if a `disasm.<ext>` asset exists in `spec/assets/<language>/`.

### boxes.yml Special Keys

Within each language entry in boxes.yml:
- `_name` — display name of the language
- `_link` — official language URL
- `_disabled` — exclude from builds
- `_disasm` — enable disassembly support

### Testing

Tests in [spec/boxes_spec.rb](spec/boxes_spec.rb) pull test assets from `spec/assets/<language>/` and run Docker containers. Each test:
1. Runs the hello world program and expects "Hello, World!" output
2. Runs the cat program with random stdin and expects it back (if cat asset exists)
3. Tests disassembly if a disasm asset and `_disasm: true` exist

### Publishing

After building, Docker Hub READMEs are updated via `ruby push.rb` (requires a `TOKEN` env var with a Docker Hub JWT). The README content is generated from [box_readme.md.erb](box_readme.md.erb) and pulls Hello World examples from the test assets.
